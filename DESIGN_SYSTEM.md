# DESIGN_SYSTEM.md — Кодекс UI

Source of truth. Все страницы приложения «Кодекс» должны соответствовать этому документу.

**ВАЖНО (Pavel 2026-05-21 ревизия):** Эталона в проекте НЕТ ни одной страницы — даже Утренний брифинг и Оглавление сделаны с косяками (serif в заголовках, эмодзи, гигантские h1 для подзаголовков, литеральные `---`, мета-инфа как основной контент, гигантские отступы, узкие карточки). Игнорируй любые установки «равняйся на Брифинг/Оглавление» из прошлых сессий.

**Визуальный эталон:** интерфейс **claude.ai** (в браузере). Открой его и используй как референс — один шрифт без засечек, минимализм, никаких эмодзи, никаких чёрных капсул, спокойный фиолетовый акцент только на active state.

Если этот документ расходится с реальным кодом — правда в документе, исправь код. (Раньше было наоборот — больше нет.)

## 1. Design tokens

### 1.1. Цвета

```css

  --color-bg-app:        #F4F4F6;   /* фон всего приложения, очень светло-серый */

  --color-bg-subtle:     #F7F7F9;   /* фон вторичных блоков, выделенных цитат */
  --color-bg-muted:      #EEEEF2;   /* фон неактивных pill-кнопок */

  /* Text */
  --color-text-primary:  #0B0B0F;   /* основной текст, заголовки */
  --color-text-secondary:#3F3F46;   /* вторичный текст */
```

## 2. Структура страницы

Все страницы рендерятся внутри `<AppShell>`:

```
│ │         │ │  PageHeader                              │ │
│ │ Sidebar │ │  ─────────────────────────────────────   │ │
│ │  260px  │ │  StatusBar / TabBar (опционально)        │ │
│ │         │ │                                          │ │
│ │         │ │  ┌────────────────────────────────────┐  │ │
│ │         │ │  │ Card                               │  │ │
│ │         │ │  │                                    │  │ │

│ │         │ │  │                                    │  │ │
│ │         │ │  └────────────────────────────────────┘  │ │
│ │         │ │                                          │ │
│ │         │ │  ┌────────────────────────────────────┐  │ │
```

**Запрещено:**

- Дублировать сайдбар в каждой странице.
- Делать страницы без `<PageHeader>`.
- Размещать контент напрямую на `--color-bg-app` без обёртки в `<Card>` (исключения — табы и фильтры над карточками).

## 3. Компоненты

### 3.1. `<Sidebar>`

- Ширина: `var(--sidebar-width)` (260px).
- Фон: `var(--color-bg-surface)`.
- Граница справа: `1px solid var(--color-border)`.
- Внутренний отступ: `var(--space-lg)`.
- Сверху — логотип-плашка: квадрат 40×40, `--radius-md`, фон `--color-accent`, белая буква «К» внутри. Справа от логотипа — название «Кодекс» (font-weight 600) и подпись «Pavel · Хилингод» (`--color-text-muted`, `--font-size-small`).
- Список пунктов меню: вертикальный, между пунктами отступ `var(--space-xs)`.
- Пункт меню:
  - padding: `var(--space-sm) var(--space-md)`.
  - `--radius-md`.
  - В обычном состоянии: текст `--color-text-secondary`, фон прозрачный.
  - В hover: фон `--color-bg-muted`.
  - В активном состоянии: фон `--color-accent-soft`, текст `--color-accent-text`, font-weight 500.
- Внизу сайдбара (прижато к низу): пункт «Компоненты» в `--color-text-muted`, `--font-size-small`.

### 3.2. `<PageHeader>`

- Структура: заголовок слева, мета-информация справа.
- Заголовок: `--font-size-h1`, font-weight 700, `--color-text-primary`.
- Подзаголовок (если есть): под заголовком, `--font-size-body`, `--color-text-muted`.
- Мета-блок справа: `--font-size-small`, `--color-text-muted`.
- Отступ снизу до контента: `var(--space-xl)`.
- Под заголовком — тонкий разделитель `1px solid var(--color-border)`, отступ до него `var(--space-md)`.

### 3.3. `<Card>`

- Фон: `var(--color-bg-surface)`.
- Скругление: `var(--radius-lg)` (16px).
- Тень: `var(--shadow-md)`.
- Внутренний отступ: `var(--space-xl)` (32px) на десктопе.
- Между карточками — отступ `var(--space-lg)`.
- Без border'а (тень даёт достаточно отделения).

### 3.4. `<Button>`

Три варианта:

**Primary**
- Фон: `--color-accent`.
- Текст: белый.
- Hover: фон `--color-accent-hover`.
- padding: `10px 20px`.
- `--radius-md`.
- font-weight 500.

**Secondary**
- Фон: `--color-bg-surface`.
- Текст: `--color-text-primary`.
- Граница: `1px solid var(--color-border)`.
- Hover: фон `--color-bg-muted`.
- Остальное — как primary.

**Ghost**
- Фон: прозрачный.
- Текст: `--color-text-secondary`.
- Hover: фон `--color-bg-muted`.

### 3.5. `<PillTabs>` / Фильтры

- Контейнер — горизонтальная группа pill-кнопок.
- Каждая pill:
  - padding: `8px 16px`.
  - `--radius-full`.
  - `--font-size-small`, font-weight 500.
  - Неактивная: фон `--color-bg-surface`, текст `--color-text-secondary`, граница `1px solid var(--color-border)`.
  - Активная: фон `--color-accent`, текст белый, без границы.

### 3.6. `<TabBar>` (табы под заголовком)

- Горизонтальный список.
- Каждый таб:
  - padding: `var(--space-sm) var(--space-md)`.
  - `--font-size-body`, font-weight 500.
  - Неактивный: `--color-text-muted`, фон прозрачный.
  - Активный: `--color-text-primary`, фон `--color-bg-surface`, тень `--shadow-sm`, `--radius-md`.

### 3.7. `<StatusBar>` (полоска статуса как «Система живая · idle_keeper · …»)

- Горизонтальная полоса между PageHeader и контентом.
- Элементы разделены отступом `var(--space-xl)`.
- Шрифт `--font-size-small`.
- Метки (`idle_keeper:`, `Pavel простой:`) — `--color-text-muted`.
- Значения — `--color-text-primary`, font-weight 500.
- Статусная точка слева — 8×8 круг, цвет `--color-success` для «живой», `--color-warning` для предупреждений, `--color-danger` для ошибок.

### 3.8. Бейджи

- padding: `2px 10px`.
- `--radius-full`.
- `--font-size-caption`, font-weight 500.
- Зелёный (покрытие, активный): граница `1px solid var(--color-success)`, текст `--color-success`, фон прозрачный.
- Серый (нейтральный): фон `--color-bg-muted`, текст `--color-text-secondary`.

### 3.9. Контент главы (длинный текст)

- Обёрнут в `<Card>`.
- Внутри карточки ширина текста ограничена `var(--reading-max-width)` (780px) с центрированием.
- Параграфы: `--font-size-body`, `line-height: var(--line-height-relaxed)`, отступ снизу `var(--space-md)`.
- Заголовки разделов (H3): отступ сверху `var(--space-xl)`, снизу `var(--space-md)`.
- Цитаты / выделенные блоки: фон `--color-bg-subtle`, padding `var(--space-md)`, `--radius-md`, левая граница `3px solid --color-accent`.

## 4. Правила работы с дизайном

1. Никогда не хардкодь цвет/радиус/отступ в компоненте. Только через переменные.
2. Никогда не вводи новый токен, не добавив его сначала в этот файл.
3. Никогда не верстай страницу «с нуля», если для блока уже есть компонент. Найди компонент и используй его.
4. Inline-стили (`style={{...}}`) запрещены везде, кроме случаев динамических значений (ширина прогресс-бара, цвет графика по данным и т.п.).
5. Перед коммитом UI-изменений — пройди чек-лист в `UI_FIX_BRIEF.md` (шаг 5).
6. Страница `Компоненты` — обновляется при любом изменении дизайн-системы.

## 5. Чек-лист соответствия (для любой страницы)

- [ ] Использует общий `<AppShell>` и `<Sidebar>`.
- [ ] Имеет `<PageHeader>`.
- [ ] Контент обёрнут в `<Card>` (или несколько `<Card>`).
- [ ] Все цвета — из токенов.
- [ ] Все радиусы — из токенов.
- [ ] Все отступы — из spacing-шкалы (`--space-*`).
- [ ] Все кнопки — компонент `<Button>` с правильным `variant`.
- [ ] Шрифт — `var(--font-sans)`, размеры — из шкалы.
- [ ] Нет inline-стилей с хардкод-значениями.
- [ ] Длинный текст — ширина ≤ `--reading-max-width`.
- [ ] Визуально неотличима от `Утренний брифинг` и `Оглавление` по: сайдбару, шапке, карточкам, кнопкам, фильтрам.
