## 4. Главные паттерны (вычисленные правила правки)

### Правило 1. Убирай связки-бюрократизмы
- `является` → удаляй или замени на `есть`
- `представляет собой` → удаляй или замени на `есть`
- `остаётся` → удаляй
- `имеет` → переформулируй активным голосом

### Правило 2. Убирай AI-каузальность «создавая/создаёт/происходит»
- AI: «Церемония создаёт пространство связи»  
  Pavel: «Церемония есть пространство связи» / «В церемонии — связь»
- AI: «Это происходит благодаря фракталу»  
  Pavel: «Это случается через фрактал» / «Фрактал делает это»

### Правило 3. Убирай ложные коннекторы
- `через` (cut 211 раз!) — Pavel удаляет где можно. AI злоупотребляет «через X»: «через церемонию», «через служение». Pavel пишет напрямую.

### Правило 4. Убирай хеджи уверенности
- `могут`, `может быть`, `возможно`, `исключительно`, `полностью` — это AI-страхование. Pavel пишет declarative: «есть», «делают», «надлежит».

### Правило 5. Сакрализуй язык
- Добавляй «священный/-ая/-ое/-ые» где уместно
- Используй «Великий Дух», «Великое Учение», «Великая Тайна»
- Вместо «должен» — `надлежит` (более формальный, сакральный)

### Правило 6. Замени роли терминами Pavel-а
- AI: «наставник», «учитель», «лидер»  
  Pavel: `Проводник` (заглавная буква, единый термин)
- AI: «член», «адепт», «практикующий»  
  Pavel: `Участник` (нейтрально и каноно)

### Правило 7. Структурируй короче
- Pavel разбивает длинные AI-параграфы на 2-3 более коротких
- Каждая мысль — отдельный абзац (ритм мантры, не лекции)

## 6. Как использовать этот канон

### Для агента (программно)
- POST `/api/synergy-edit` с `{text, chapter_context?}` → возвращает Pavel-style версию
- Серверная сторона включает канон в system-prompt и зовёт Sonnet
- Применяй автоматически ПОСЛЕ любой AI-генерации перед показом Pavel-у

### Для человека-редактора
1. Прогони текст через чёрный/белый списки (поиск-замена)
2. Перечитай с правилами §4 в голове
3. Сократи длинные параграфы на 2-3 коротких
4. Сверь финал с эталонными примерами §5
